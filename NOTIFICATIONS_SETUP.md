# نظام الإشعارات - إعداد وتوثيق

## ✅ الميزات المضافة

تم إضافة نظام إشعارات كامل يسمح للمدير بإرسال إشعارات للمستخدمين.

### الميزات:
1. ✅ قسم "🔔 الإشعارات" في لوحة التحكم
2. ✅ إرسال إشعارات لجميع المستخدمين أو مستخدم محدد
3. ✅ أنواع مختلفة من الإشعارات (معلومات، نجاح، تحذير، خطأ)
4. ✅ عرض الإشعارات في نافذة الإشعارات للمستخدمين
5. ✅ تحديث فوري للإشعارات (Real-time)
6. ✅ سجل الإشعارات المُرسلة في لوحة التحكم

## 🗄️ إعداد قاعدة البيانات

### إنشاء جدول notifications

افتح Supabase SQL Editor وقم بتشغيل:

```sql
-- إنشاء جدول الإشعارات
CREATE TABLE notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  type TEXT NOT NULL DEFAULT 'info',
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  read BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- إنشاء فهرس للأداء
CREATE INDEX idx_notifications_user ON notifications(user_id);
CREATE INDEX idx_notifications_read ON notifications(read);
CREATE INDEX idx_notifications_created ON notifications(created_at);

-- تفعيل RLS (Row Level Security)
ALTER TABLE notifications ENABLE ROW LEVEL SECURITY;

-- Policy للسماح للمستخدمين بقراءة إشعاراتهم فقط
CREATE POLICY "Users can read own notifications"
ON notifications FOR SELECT
USING (auth.uid()::text = user_id::text);

-- Policy للسماح للمديرين بإضافة إشعارات
CREATE POLICY "Admins can insert notifications"
ON notifications FOR INSERT
WITH CHECK (true);

-- Policy للسماح للمستخدمين بتحديث حالة القراءة
CREATE POLICY "Users can update own notifications"
ON notifications FOR UPDATE
USING (auth.uid()::text = user_id::text);
```

## 📋 كيفية الاستخدام

### للمديرين:

#### 1. إرسال إشعار جديد:
1. ادخل إلى لوحة التحكم
2. اضغط على تبويب "🔔 الإشعارات"
3. املأ البيانات:
   - **إرسال إلى:** اختر "جميع المستخدمين" أو مستخدم محدد
   - **نوع الإشعار:** اختر (معلومات، نجاح، تحذير، خطأ)
   - **عنوان الإشعار:** اكتب العنوان
   - **محتوى الإشعار:** اكتب المحتوى
4. اضغط "📤 إرسال الإشعار الآن"

#### 2. عرض سجل الإشعارات:
- في نفس القسم، ستجد جدول بجميع الإشعارات المُرسلة
- يعرض: المستخدم، النوع، العنوان، وقت الإرسال، الحالة

### للمستخدمين:

#### 1. استقبال الإشعارات:
- تظهر الإشعارات تلقائياً في نافذة الإشعارات (🔔)
- يتم تحديثها فوراً عند إرسالها من المدير
- يتم تحديثها كل 8 ثوانٍ تلقائياً

#### 2. عرض الإشعارات:
- اضغط على أيقونة الجرس (🔔) في شريط التنقل
- ستظهر نافذة بجميع الإشعارات غير المقروءة
- عند فتح النافذة، يتم تحديد الإشعارات كمقروءة تلقائياً

## 📊 البنية التقنية

### الجدول notifications:
```sql
- id: UUID (Primary Key)
- user_id: UUID (Foreign Key → users)
- type: TEXT (info, success, warning, error)
- title: TEXT
- content: TEXT
- read: BOOLEAN (default: false)
- created_at: TIMESTAMPTZ
```

### الوظائف الرئيسية:
- `sendNotification()` - إرسال إشعار
- `loadNotif()` - تحميل سجل الإشعارات
- `loadNotifUsers()` - تحميل قائمة المستخدمين
- `loadUserNotifications()` - تحميل إشعارات المستخدم
- `markNotificationsRead()` - تحديد كمقروءة
- `addN()` - إضافة إشعار للنافذة (محدثة)
- `renderNP()` - عرض الإشعارات (محدثة)

## ⚠️ ملاحظات مهمة

1. **الإشعارات تُحفظ في قاعدة البيانات** - لا تُفقد عند تحديث الصفحة
2. **Real-time check كل 8 ثوانٍ** - قد يستغرق ظهور الإشعار بضع ثوانٍ
3. **الإشعارات تُحدد كمقروءة** عند فتح نافذة الإشعارات
4. **يمكن إرسال إشعارات لجميع المستخدمين** أو مستخدم محدد
5. **الإشعارات القديمة** تبقى في السجل ولكن لا تظهر في النافذة بعد القراءة
