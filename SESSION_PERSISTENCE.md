# نظام حفظ الجلسة (Session Persistence)

## ✅ التحديثات المضافة

تم إضافة نظام حفظ الجلسة باستخدام `localStorage` ليبقى المستخدم مسجلاً حتى بعد تحديث الصفحة.

## 🔧 الوظائف المضافة

### 1. `saveSession()`
```javascript
// حفظ بيانات الجلسة في localStorage
function saveSession(){
  if(ME){
    localStorage.setItem('trendyol_user',JSON.stringify({id:ME.id,email:ME.email}));
    localStorage.setItem('trendyol_type','user');
  }else if(ADM){
    localStorage.setItem('trendyol_type','admin');
  }
}
```

**الوظيفة:**
- يحفظ بيانات المستخدم (ID و Email) عند تسجيل الدخول
- يحفظ نوع الجلسة (user أو admin)

### 2. `restoreSession()`
```javascript
// استعادة الجلسة من localStorage
async function restoreSession(){
  // يقرأ البيانات المحفوظة
  // يتحقق من صحة الحساب
  // يستعيد حالة المستخدم
}
```

**الوظيفة:**
- يقرأ البيانات من localStorage عند تحميل الصفحة
- يتحقق من وجود المستخدم في قاعدة البيانات
- يتحقق من أن الحساب غير محظور
- يستعيد حالة المستخدم تلقائياً

### 3. `clearSession()`
```javascript
// مسح الجلسة من localStorage
function clearSession(){
  localStorage.removeItem('trendyol_user');
  localStorage.removeItem('trendyol_type');
  ME=null;
  ADM=false;
}
```

**الوظيفة:**
- يمسح جميع بيانات الجلسة من localStorage
- يعيد تعيين المتغيرات

## 🔄 التعديلات على الوظائف الموجودة

### 1. `doLogin()`
```javascript
// تمت إضافة: saveSession()
ME=r.data;ADM=false;
saveSession(); // حفظ الجلسة
```

### 2. `doAdminLogin()`
```javascript
// تمت إضافة: saveSession()
ADM=true;ME=null;
saveSession(); // حفظ الجلسة
```

### 3. `doRegister()`
```javascript
// تمت إضافة: saveSession()
ME=r.data;ADM=false;
saveSession(); // حفظ الجلسة
```

### 4. `doLogout()`
```javascript
// تمت إضافة: clearSession()
clearSession(); // مسح الجلسة من localStorage
```

### 5. `init()`
```javascript
// تمت إضافة: استعادة الجلسة عند تحميل الصفحة
var restored=await restoreSession();
if(!restored){
  // إذا لم تكن هناك جلسة محفوظة، يعرض الصفحة العادية
}
```

## 📋 كيفية العمل

### عند تسجيل الدخول:
1. المستخدم يسجل دخوله
2. يتم حفظ بياناته في `localStorage`
3. يتم حفظ نوع الجلسة (user/admin)

### عند تحديث الصفحة:
1. يتم استدعاء `init()` تلقائياً
2. يتم استدعاء `restoreSession()`
3. يتم قراءة البيانات من `localStorage`
4. يتم التحقق من صحة الحساب في قاعدة البيانات
5. إذا كان الحساب صحيحاً، يتم استعادة الجلسة تلقائياً
6. يتم توجيه المستخدم للصفحة المناسبة

### عند تسجيل الخروج:
1. يتم مسح جميع البيانات من `localStorage`
2. يتم إعادة تعيين المتغيرات
3. يتم توجيه المستخدم لصفحة الهبوط

## 🔒 الأمان

### التحقق من الحساب:
- يتم التحقق من وجود المستخدم في قاعدة البيانات عند كل استعادة
- يتم التحقق من أن الحساب غير محظور
- إذا كان الحساب محظوراً، يتم مسح الجلسة تلقائياً

### البيانات المحفوظة:
- **للمستخدمين:** ID و Email فقط (لا يتم حفظ كلمة المرور)
- **للمديرين:** نوع الجلسة فقط

## ⚠️ ملاحظات مهمة

1. **لا يتم حفظ كلمة المرور** في localStorage
2. **يتم التحقق من الحساب** في قاعدة البيانات عند كل استعادة
3. **إذا تم حظر الحساب**، يتم مسح الجلسة تلقائياً
4. **يمكن للمستخدم تسجيل الخروج يدوياً** لمسح الجلسة

## 🧪 الاختبار

### اختبار حفظ الجلسة:
1. سجّل دخولك
2. حدّث الصفحة (F5)
3. يجب أن تبقى مسجلاً

### اختبار مسح الجلسة:
1. سجّل دخولك
2. اضغط "خروج"
3. حدّث الصفحة (F5)
4. يجب أن تكون غير مسجل

### اختبار الحساب المحظور:
1. سجّل دخولك
2. احظر الحساب من لوحة التحكم
3. حدّث الصفحة (F5)
4. يجب أن يتم تسجيل الخروج تلقائياً

## 🔄 التحديثات المستقبلية (اختيارية)

يمكن إضافة:
- **انتهاء صلاحية الجلسة** بعد فترة زمنية معينة
- **تشفير البيانات** المحفوظة في localStorage
- **تحديث تلقائي** لبيانات المستخدم من قاعدة البيانات
- **إشعار** عند انتهاء الجلسة
