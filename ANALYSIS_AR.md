## الجزء الأول: أداة تحويل SVG إلى Java

### الهدف
تحويل ملفات SVG إلى كود Java جاهز للاستخدام في تطبيقات Android بدون الحاجة لمكتبات خارجية.

### البنية الأساسية

#### 1. **Main.java** - نقطة البداية
```java
// الوظيفة الرئيسية:
- قراءة ملف SVG كمدخل
- تحليل الملف باستخدام مكتبة AndroidSVG
- تحويل عمليات الرسم إلى كود Java
- حفظ الملف الناتج كـ .java
```

**المعاملات المدعومة:**
- `-p`: اسم الحزمة (package)
- `-c`: اسم الكلاس
- `-o`: اسم ملف الإخراج
- `-t`: ملف قالب مخصص
- `-aos`: الحد الأدنى لإصدار Android API
- `-rt`: استبدال النصوص
- `-rc`: استبدال الألوان

#### 2. **OutputBuilder.java** - بناء الكود
```java
// الوظائف الرئيسية:
- جمع عمليات الرسم في قوائم منظمة
- تقسيم الكود إلى دوال فرعية إذا كان كبيراً
- تحسين الكود (إزالة save/restore غير الضرورية)
- دمج الكود النهائي في ملف واحد
```

**آلية التقسيم:**
- إذا تجاوز الكود 1500 سطر، يتم تقسيمه إلى دوال فرعية
- كل دالة فرعية تسمى `render_N()` حيث N رقم متسلسل

#### 3. **AndroidClass.java** - محاكاة كلاسات Android
```java
// الوظيفة:
- تتبع أسماء الكائنات المُنشأة
- إدارة التبعيات بين الكائنات
- إزالة الكائنات غير المستخدمة
```

#### 4. **SVGAndroidRenderer.java** - محرك الرسم
```java
// الوظيفة:
- تحليل عناصر SVG (paths, shapes, text, gradients)
- تحويل كل عنصر إلى أوامر Canvas في Android
- إدارة الحالة (colors, transforms, styles)
```

### سير العمل (Workflow)

```
1. قراءة SVG → SVG.getFromInputStream()
2. تحليل SVG → SVG.parse()
3. محاكاة الرسم → SVGAndroidRenderer.render()
4. جمع الأوامر → OutputBuilder.append()
5. تحسين الكود → OutputBuilder.optimize()
6. تقسيم الدوال → OutputBuilder.splitMethod()
7. إنشاء الملف → OutputBuilder.getResult()
```

## الجزء الثاني: منصة Trendyol الاستثمارية

### الهدف
منصة استثمار إلكترونية تتيح للمستخدمين:
- استقبال أكواد ربح يومية
- إيداع وسحب الأموال
- نظام إحالة
- لوحة تحكم إدارية

### البنية التقنية

#### 1. **قاعدة البيانات (Supabase)**
```javascript
// الجداول المستخدمة:

- codes_log: سجل الأكواد المُرسلة
- transactions: سجل المعاملات
- deposits: طلبات الإيداع
- withdrawals: طلبات السحب
- settings: إعدادات النظام
- admins: بيانات المديرين
- schedule_log: سجل الإرسال التلقائي
```

#### 2. **الصفحات الرئيسية**

##### أ. **صفحة الهبوط (Landing)**
```javascript
// الوظائف:
- عرض معلومات المنصة
- زر التسجيل والدخول
- عرض مزايا النظام
- عرض نظام الإحالة (8 أشخاص = $1,000)
```

##### ب. **صفحة تسجيل الدخول (Login)**
```javascript

  // التحقق من البريد وكلمة المرور
  // تحديث حالة المستخدم
```

##### ج. **صفحة التسجيل (Register)**

##### د. **لوحة تحكم المستخدم (User Dashboard)**

**الأقسام:**
1. **الرئيسية (Home)**
   - عرض الرصيد الكلي
   - عرض رصيد الأرباح
   - عرض الأرباح المعلقة
   - عرض أكواد اليوم

   - تفعيل أكواد جديدة

3. **المحفظة (Wallet)**

   - رصيد الأرباح
   - المكافآت

4. **الإحالات (Referrals)**
   - كود الإحالة الخاص
   - رابط الدعوة
   - عدد الإحالات المؤهلة (8 أشخاص × $500)

5. **السجل (History)**
   - جميع المعاملات

6. **التوثيق (KYC)**
   - رقم الهوية
   - حالة التوثيق

4. يرسل المبلغ
5. يرسل صورة المعاملة للدعم

## نقاط مهمة في الكود

### 1. **الأمان**
- كلمات المرور مخزنة كنص عادي (⚠️ غير آمن)
- لا يوجد تشفير للبيانات الحساسة
- التحقق من الحساب يتم عبر status field

### 2. **الأداء**
- استخدام Supabase Real-time (لكن غير مفعّل في الكود)
- فحص دوري كل 8 ثوانٍ (polling)
- تحميل البيانات عند الحاجة (lazy loading)

### 3. **التصميم**
- تصميم متجاوب (responsive)
- دعم اللغة العربية (RTL)
- واجهة مستخدم حديثة

### 4. **قاعدة البيانات**
```sql
-- الجداول الرئيسية:
users: id, name, email, password, balance, profit_bal, pending_profit, bonus, ref_code, ref_by, status, kyc_id, kyc_status
codes_log: id, user_id, code, profit_pct, used, used_at, created_at, slot
transactions: id, user_id, type, amount, network, status, created_at
deposits: id, user_id, user_name, amount, network, status, created_at
withdrawals: id, user_id, user_name, amount, wallet, network, status, created_at
settings: id, tron_wallet, eth_wallet, support_email, withdraw_fee, min_withdraw
admins: id, username, password
schedule_log: id, date, slot, sent_at
```
